# 技能结构规范

本文档描述了 agent-skills 技能文件的结构和格式。在贡献新技能或理解现有技能时，请以此为指南。

## 文件位置

每个技能位于 `skills/` 下的独立目录中：

```
skills/
  skill-name/
    SKILL.md          # 必需：技能定义
    supporting-file.md # 可选：按需加载的参考资料
```

## SKILL.md 格式

### Frontmatter（必需）

```yaml
---
name: skill-name-with-hyphens
description: Brief statement of what the skill does. Use when [specific trigger conditions].
---
```

**规则：**
- `name`：小写，连字符分隔。必须与目录名匹配。
- `description`：以技能功能开头（第三人称），后跟触发条件。需包含"做什么"和"何时用"。最长 1024 个字符。

**为什么重要：** 代理通过读取描述来发现技能。描述会被注入系统提示词，因此必须告诉代理技能提供了什么以及何时激活它。不要总结工作流 -- 如果描述包含流程步骤，代理可能会按摘要执行而非阅读完整技能。

### 标准章节

```markdown
# 技能标题

## 概述
一两句话解释该技能的功能和重要性。

## 何时使用
- 触发条件的列表（症状、任务类型）
- 何时不使用（排除项）

## [核心流程 / 工作流 / 步骤]
主要工作流，分为编号步骤或阶段。
在有助于理解的地方包含代码示例。
在存在决策点的地方使用流程图（ASCII）。

## [特定技术 / 模式]
针对特定场景的详细指导。
代码示例、模板、配置。

## 常见借口
| 借口 | 现实 |
|---|---|
| 代理用来跳过步骤的借口 | 为什么这个借口是错的 |

## 红色警告
- 表明技能被违反的行为模式
- 审查时需要注意的事项

## 验证
完成技能流程后，确认：
- [ ] 退出标准检查清单
- [ ] 证据要求
```

## 章节用途

### 概述
技能的"电梯演讲"。应回答：该技能做什么，为什么代理应该遵循它？

### 何时使用
帮助代理和人类判断该技能是否适用于当前任务。包含正向触发条件（"当 X 时使用"）和负向排除项（"不适用于 Y"）。

### 核心流程
技能的核心。这是代理遵循的分步工作流。必须具体且可操作 -- 不是含糊的建议。

**好的写法：** "运行 `npm test` 并验证所有测试通过"
**差的写法：** "确保测试能工作"

### 常见借口
精心设计的技能中最具特色的部分。这些是代理用来跳过重要步骤的借口，配以反驳。它们防止代理为自己找理由来绕过流程。

想想代理说过多少次"我稍后添加测试"或"这个足够简单可以跳过规格说明" -- 这些都应放在此处，并附上事实性的反驳论点。

### 红色警告
技能被违反的可观察信号。在代码审查和自我监控时有用。

### 验证
退出标准。代理用于确认技能流程已完成的检查清单。每个复选框都应该可以用证据验证（测试输出、构建结果、截图等）。

## 支持文件

仅在以下情况创建支持文件：
- 参考资料超过 100 行（保持主 SKILL.md 聚焦）
- 需要代码工具或脚本
- 检查清单足够长，需要独立文件

不足 50 行的模式和原则保持内联。

## 写作原则

1. **流程优于知识。** 技能是工作流，不是参考文档。步骤，而非事实。
2. **具体优于笼统。** "运行 `npm test`" 优于 "验证测试"。
3. **证据优于假设。** 每个验证复选框都需要证明。
4. **反借口化。** 每个可能被跳过的步骤都需要在借口表中有对应的反驳。
5. **渐进式披露。** 主 SKILL.md 是入口。支持文件仅在需要时加载。
6. **注意 token 消耗。** 每个章节都必须证明其存在价值。如果移除它不会改变代理行为，就移除它。

## 命名约定

- 技能目录：`lowercase-hyphen-separated`（小写连字符分隔）
- 技能文件：`SKILL.md`（始终大写）
- 支持文件：`lowercase-hyphen-separated.md`（小写连字符分隔）
- 参考资料：存放在项目根目录的 `references/` 中，不在技能目录内

## 跨技能引用

通过名称引用其他技能：

```markdown
按照 `test-driven-development` 技能编写测试。
如果构建失败，使用 `debugging-and-error-recovery` 技能。
```

不要在技能之间复制内容 -- 使用引用和链接代替。
